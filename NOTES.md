Στον κόσμο του React (και του frontend γενικότερα), State (Κατάσταση) είναι τα δεδομένα που "θυμάται" η εφαρμογή σου σε μια συγκεκριμένη χρονική στιγμή.

Το State Management είναι η στρατηγική και τα εργαλεία που χρησιμοποιούμε για να διαχειριστούμε αυτά τα δεδομένα, ειδικά όταν η εφαρμογή μεγαλώνει και πολλά διαφορετικά μέρη της (components) πρέπει να μοιράζονται τις ίδιες πληροφορίες.

## 1. Γιατί το χρειάζεσαι στο δικό σου Project;
Σκέψου το Dashboard που φτιάχνεις:

User State: Το όνομα και το avatar του προγραμματιστή που έκανε login. Πρέπει να εμφανίζεται στο sidebar αλλά και μέσα στο chat.

Project State: Ποιο project είναι επιλεγμένο αυτή τη στιγμή; Αν αλλάξεις project, πρέπει να ενημερωθούν ταυτόχρονα τα γραφήματα, το wiki και το chat.

Chat State: Το ιστορικό των μηνυμάτων που ανταλλάσσεις με το bot.

Χωρίς State Management, θα έπρεπε να "πασάρεις" αυτές τις πληροφορίες από τον πατέρα στο παιδί (Prop Drilling), κάτι που σε ένα σύνθετο dashboard καταλήγει σε χάος.
